### 問題1: サイドバーエラー「知識ベースが読み込めません」

**原因:**
`knowledge_base/` ディレクトリまたは `qa_knowledge_base_all.json` が正しく配置されていない

**解決方法:**

#### ステップ1: 知識ベース配置確認スクリプトを実行

```bash
python check_kb.py
```

このスクリプトが以下を確認します:
- 現在の作業ディレクトリ
- app.pyの場所
- knowledge_base/ ディレクトリの存在
- qa_knowledge_base_all.json の存在
- JSONファイルの読み込み可否

#### ステップ2: 必要に応じてディレクトリとファイルを配置

check_kb.py の出力に従って、以下を実行:

```bash
# プロジェクトのルートディレクトリに移動（app.pyがある場所）
cd /path/to/your/project

# knowledge_baseディレクトリを作成
mkdir -p knowledge_base

# qa_knowledge_base_all.json をコピー
cp /path/to/downloads/qa_knowledge_base_all.json knowledge_base/
```

#### ステップ3: 再度確認

```bash
python check_kb.py
```

期待される出力:
```
✅ 読み込み成功: 93件のQ&A
✅ 正常に読み込めます!
```

## 🔍 トラブルシューティング

### エラー: 知識ベースが読み込めません

**確認事項:**

1. **ディレクトリ構成**
   ```
   your_project/
   ├── app.py
   ├── knowledge_base/         ← このディレクトリがあるか
   │   └── qa_knowledge_base_all.json  ← このファイルがあるか
   └── check_kb.py
   ```

2. **ファイルパスの確認**
   ```bash
   ls -la knowledge_base/qa_knowledge_base_all.json
   ```

3. **JSONファイルの内容確認**
   ```bash
   python -c "import json; print(len(json.load(open('knowledge_base/qa_knowledge_base_all.json'))['qa_pairs']))"
   ```
   
   期待される出力: `93`

### エラー: モジュールがない

```bash
pip install streamlit openai python-dotenv pandas matplotlib
```

### エラー: BASE_DIRが正しくない

app.pyと同じディレクトリで起動しているか確認:

```bash
# 正しい
cd /path/to/project  # app.pyがある場所
streamlit run app.py

# 間違い
cd /some/other/directory
streamlit run /path/to/project/app.py  # ← BASE_DIRが正しく設定されない
```
