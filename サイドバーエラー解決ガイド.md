# 🚨 サイドバーエラー「知識ベースが読み込めません」の解決方法

## 現在の状況
サイドバーに「❌ 知識ベースが読み込めません」と表示されている

## 原因
`knowledge_base/qa_knowledge_base_all.json` ファイルが正しく配置されていない

---

## ✅ 解決方法（3ステップ）

### ステップ1: app.pyがある場所に移動

```bash
# ターミナルで、app.pyがあるディレクトリに移動
cd /path/to/your/project
# 例: cd ~/Documents/mahoboard
```

確認方法:
```bash
ls app.py
# app.py が表示されればOK
```

---

### ステップ2: knowledge_baseディレクトリを作成

```bash
mkdir -p knowledge_base
```

確認方法:
```bash
ls -la
# knowledge_base/ というディレクトリが表示されればOK
```

---

### ステップ3: qa_knowledge_base_all.json を配置

ダウンロードした `qa_knowledge_base_all.json` を `knowledge_base/` にコピー:

```bash
# ダウンロードフォルダから配置する例
cp ~/Downloads/qa_knowledge_base_all.json knowledge_base/
```

確認方法:
```bash
ls -lh knowledge_base/qa_knowledge_base_all.json
# ファイルが表示され、サイズが約76KBであればOK
```

## 🔍 それでもエラーが出る場合

### デバッグ情報の確認

サイドバーのエラーメッセージの下にある「🔍 デバッグ情報」を開いて、以下を確認:

1. **作業ディレクトリ**: app.pyがある場所と一致しているか
2. **期待されるパス**: 正しいパスが表示されているか
3. **ディレクトリ存在**: ✅になっているか
4. **ファイル存在**: ✅になっているか

### よくある問題

#### 問題1: 別のディレクトリで起動している

```bash
# 間違い
cd ~/Desktop
streamlit run ~/Documents/project/app.py

# 正しい
cd ~/Documents/project
streamlit run app.py
```

#### 問題2: ファイル名が違う

```bash
# 確認
ls knowledge_base/
# qa_knowledge_base_all.json と完全に一致するか確認
# qa_knowledge_base_all (1).json などになっていないか
```

#### 問題3: ファイルパーミッション

```bash
# 読み取り権限を確認
ls -la knowledge_base/qa_knowledge_base_all.json
# -rw-r--r-- のようになっていればOK

# もし権限がなければ
chmod 644 knowledge_base/qa_knowledge_base_all.json
```
